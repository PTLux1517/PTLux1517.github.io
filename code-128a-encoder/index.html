Code 128A Encoder

Reference Info

Detailed specification: barcodefaq.com/1d/code-128

Barcode data below is case sensitive. Ordinary data must be upper case and control characters must be lower case.

Some common control characters are:

  • escape: {
  • enter: m
  • backspace: h

Example

For example, given the following navigation path

  • 8: Utilities
  • C: Create Empty Tags
  • (some field which takes a number of empty tags to create as input)
  • (user input followed by the enter key, followed by the Y key to confirm)

one would enter the following data to create 10 empty tags:

{{{{8C10mY{

The first four {'s are translated to escape key presses, backing out of whatever screen the barcode reader is currently on to the main menu.

The 8 navigates from the main menu to the "Utilities" screen.

The C navigates from the "Utilities" screen to the "Create Empty Tags" screen.

The 10 populates the field with the number of tags to create.

The m translates to an enter key press. (note that it's lower case)

The Y is simply an ordinary Y key press, confirming the creation of 10 tags. (note that it's upper case)

The final { translates to an escape key press, backing out of the "Create Empty Tags" screen to the "Utilities" screen.

At the end of the barcode, the barcode scanner automatically issues the enter key, and since the "Create Empty Tags" menu option is currently highlighted, the automatic enter key press navigates back into the "Create Empty Tags" screen.

Use font Libre Barcode 128 or another equivalent to render the encoded barcode elsewhere.
